# Rectitude.AI Technical Architecture Report

## 🚀 Core Technology Stack

| Technology | Role | Rationale |
| :--- | :--- | :--- |
| **FastAPI** | Backend Framework | High-performance async processing with built-in schema validation. |
| **Ollama** | LLM Engine | Ensures 100% data privacy by keeping all inference local. |
| **Mistral 7B** | Primary LLM | industry-leading intelligence-to-weight ratio for local execution. |
| **SQLite** | Database | Lightweight and portable persistence for demo HR/Financial data. |
| **RestrictedPython** | Security Sandbox | Hardened runtime environment for executing untrusted AI code. |

## 🛡️ Security Layers (Defense-in-Depth)

1. **Layer 1: Intent Defense (Regex)**
   - Pre-filters prompts for "Ignore Instructions", "Config Probing", and "Role Hijacking".
   - Latency: <5ms.

2. **Layer 2: Capability Control (RBAC)**
   - Agents are restricted by "Capability Tokens".
   - An HR agent physically cannot access the Email Tool, even if jailbroken.

3. **Layer 3: Output Mediation (Redaction)**
   - Real-time PII and Salary detection.
   - Automatically redacts sensitive patterns before they reach the user.

4. **Layer 4: Sandbox (Code Isolation)**
   - All Python code generated by the AI runs in an isolated environment with no access to the host file system or network.
